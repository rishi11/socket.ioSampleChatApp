Socket Io Sample App

  • The data of the application is volatile. There is no DB used.
  • This is sample APP demonstarating the use of sockets using Socket.io.
  • To enter chat room, you have to provide a distinct user EVERYTIME.
  • The username is not fixed, you might not get same username everytime.
  • The session is maintained by storing userID in Angular rootScope, hence refreshing the page will loose it.
  • If you close the window, the application will consider you as logged out.
  • In chat view, you might have to scroll table to last to se latest message.
